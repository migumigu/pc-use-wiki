# 上下文窗口极简主义

> 只喂给 AI 当前步骤必要的信息，保持注意力集中

<!-- confidence: EXTRACTED -->
上下文窗口极简主义是上下文工程的核心原则之一，指在 AI Agent 运行过程中，只向模型提供当前步骤必要的信息，避免信息过载导致注意力分散。

## 核心思想

### 信息拆分策略

将信息拆分到不同文件：
- **task_plan.md**：任务目标和进度
- **notes.md**：调研资料和中间想法
- **[deliverable].md**：最终输出结果

### 按需加载

AI 在执行某一步时只需要读取相关片段，而不是把所有相关文件、所有历史对话都丢给模型。

## Claude Code 的实践

Claude Code 采用「分而治之」和「按需加载」的策略，而非单纯依赖更大的上下文窗口。

## 为什么重要

1. **注意力集中**：信息越多，模型越容易分心
2. **成本控制**：Token 成本是主要瓶颈
3. **检索质量**：向量数据库检索质量不稳定
