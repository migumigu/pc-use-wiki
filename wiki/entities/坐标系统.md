# 坐标系统

> GUI自动化中基于屏幕坐标的定位技术,通过(x,y)坐标定位鼠标和UI元素

## 定义

<!-- confidence: EXTRACTED -->
坐标系统是GUI自动化中用于定位屏幕上位置的基础方法,通过(x,y)二维坐标表示鼠标位置和UI元素位置。

## 基本概念

### 屏幕坐标系
<!-- confidence: EXTRACTED -->
- **原点**: 屏幕左上角(0,0)
- **X轴**: 水平方向,向右递增
- **Y轴**: 垂直方向,向下递增
- **坐标范围**: (0,0)到(screenWidth, screenHeight)

### PyAutoGUI中的坐标
<!-- confidence: EXTRACTED -->
```python
screenWidth, screenHeight = pyautogui.size()  # 获取屏幕尺寸
currentMouseX, currentMouseY = pyautogui.position()  # 获取鼠标位置
pyautogui.moveTo(100, 150)  # 移动到指定坐标
```

## 技术挑战

### DPI缩放问题
<!-- confidence: INFERRED -->
高DPI显示器可能导致坐标映射不准确,需要处理缩放因子。

### 多显示器支持
<!-- confidence: EXTRACTED -->
PyAutoGUI不支持多显示器,仅能控制主显示器。

### 动态界面变化
<!-- confidence: INFERRED -->
窗口位置变化时,坐标定位可能失效,需要结合图像识别或UI Automation。

## 与其他定位方法的对比

### 坐标系统 vs UI Automation
<!-- confidence: INFERRED -->
- **坐标系统**: 通过位置定位,简单但易失效
- **UI Automation**: 通过元素属性定位,更稳定但复杂

### 坐标系统 vs 视觉识别
<!-- confidence: INFERRED -->
- **坐标系统**: 固定坐标,界面变化后失效
- **视觉识别**: 通过图像匹配,适应界面变化

## 在Agent控制中的应用

<!-- confidence: EXTRACTED -->
PyAutoGUI使用坐标系统作为鼠标定位的基础方法,Open Interpreter Computer Use在此基础上添加视觉识别能力。
