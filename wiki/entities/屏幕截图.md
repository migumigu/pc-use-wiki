# 屏幕截图

> 捕获屏幕图像的技术,是Computer Use和视觉Agent获取视觉信息的基础方法

## 定义

<!-- confidence: EXTRACTED -->
屏幕截图(Screen Capture)是从显示器捕获当前界面图像的技术,为Agent提供视觉信息用于分析和决策。

## 在Computer Use中的作用

### 视觉输入
<!-- confidence: EXTRACTED -->
- 为LLM提供当前界面信息
- 供OCR提取文字
- 用于图像匹配识别元素

### 截图类型
<!-- confidence: EXTRACTED -->
- **全屏截图**: 整个显示器画面
- **窗口截图**: 特定应用窗口
- **区域截图**: 指定区域范围

## 技术实现

### mss库
<!-- confidence: EXTRACTED -->
高性能屏幕截图库,Open Interpreter Computer Use的默认选择。

### pyscreenshot
<!-- confidence: EXTRACTED -->
跨平台屏幕截图库,作为mss的备选方案。

### PyAutoGUI截图
<!-- confidence: EXTRACTED -->
```python
screenshot = pyautogui.screenshot()
```

## 工作流程中的位置

<!-- confidence: EXTRACTED -->
在Computer Use工作流程中,屏幕截图是第一步"观察"的关键技术:
1. 观察(截图) → 2. 分析 → 3. 规划 → 4. 执行 → 5. 循环

## 技术挑战

### DPI缩放
<!-- confidence: EXTRACTED -->
高DPI显示器需要处理缩放因子,否则坐标映射不准确。

### 性能延迟
<!-- confidence: EXTRACTED -->
截图操作引入延迟,影响响应速度。

### 多显示器
<!-- confidence: EXTRACTED -->
部分工具(如PyAutoGUI)不支持多显示器,仅能捕获主显示器。

## 与其他视觉技术的配合

### 与OCR配合
<!-- confidence: EXTRACTED -->
截图 → OCR文字提取 → UI元素定位

### 与视觉语言模型配合
<!-- confidence: EXTRACTED -->
截图 → VLM理解 → 操作决策

### 与图像匹配配合
<!-- confidence: EXTRACTED -->
截图 → 图像匹配 → 元素识别

## 配置选项

### Open Interpreter配置
<!-- confidence: EXTRACTED -->
```python
interpreter.computer.use_screenshot = True
```

### 缩放因子处理
<!-- confidence: EXTRACTED -->
```bash
interpreter --computer-scaling-factor 1.5
```
