# agency-agents

> 122K+ Stars 的 AI Agent 人格集合库，16 个 Division 架构，51+ 专业化 Agent

## 核心定位

**agency-agents** 是一个完整的 AI Agency 技能库，核心理念是组建一支永不休息、永不抱怨、始终交付的 AI 专家团队。每个 Agent 都具备领域专精、个性驱动、交付导向、生产就绪的特征。

## Division 架构

项目采用"虚拟机构"的组织架构，将 Agent 按职能划分为 16 个 Division：

| Division | Agent 数量 | 核心职责 |
|----------|-----------|----------|
| Engineering | 34+ | Frontend/Backend/Mobile/AI/DevOps/Network/Embedded/SRE |
| Design | 9 | UI/UX/Brand/Visual/Whimsy Injector |
| Paid Media | 7 | PPC/Search Query/Tracking/Programmatic |
| Sales | 9 | Outbound/Discovery/Deal/Proposal/Pipeline |
| Security | 多个 | Security Architecture/AppSec/Pentest/Threat Intel |

## Agent 技能规范

### 标准文件结构
每个 Agent 必须遵循以下结构：
- **frontmatter**：name/description/color/emoji/vibe/services
- **Persona 层**：Identity/Memory/Communication/Rules
- **Operations 层**：Mission/Deliverables/Workflow/Metrics/Capabilities

### 设计原则
1. **Strong Personality** - 独特声音和性格
2. **Clear Deliverables** - 具体代码示例和模板
3. **Success Metrics** - 可量化指标
4. **Proven Workflows** - 经过实战验证的流程
5. **Learning Memory** - 模式识别和改进

## 工具链设计

支持 13 个平台一键安装：
- Claude Code、Cursor、Copilot、Gemini CLI、OpenCode、OpenClaw
- Antigravity、Aider、Windsurf、Kimi、Codex、Osaurus、Hermes

## 技术亮点

### Multi-Agent Systems Architect
Engineering Division 特色 Agent，专注于：
- 拓扑设计
- 上下文管理
- 信任机制
- 失败恢复

### Division 管理
- `divisions.json` 作为单一真理来源
- 标准化的贡献流程和 PR Checklist

## 能力边界

- **适合场景**：需要专业化 Agent 协作的复杂项目
- **不适用场景**：单 Agent 简单任务、非规范化工作流
