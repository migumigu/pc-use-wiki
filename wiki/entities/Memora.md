# Memora

> 微软研究院 ICML 2026 论文提出的 Agent 长期记忆系统，通过 Harmonic Memory 三重索引架构实现解耦存储与检索，上下文 token 消耗减少约 98%

## 概述

Memora 是微软研究院开发的 Agent 长期记忆系统，论文已被 ICML 2026 接收，代码于 2026 年 6 月 16 日以 MIT 协议开源。其核心创新"Harmonic Memory"通过解耦存储与检索，将记忆的丰富细节与查找的把手分离，使检索从"盲目猜测"变为"导航行为"。

Memora 的核心洞察在于"存储什么"与"如何检索"应该彻底拆开——记忆内容可以保持丰富和表现力，一个独立的轻量级结构层负责索引和检索。这打破了现有方案的两极困境：碎片化极端（RAG/Mem0，保留丰富细节但丧失叙事连贯性）与粗粒度极端（摘要压缩，效率高但剥离细节）。

在 LongMemEval 基准上，Memora 达到 87.4% 准确率（全上下文推理仅 65.6%），上下文 token 从 115,000 降至约 2,900（减少约 98%）。知识更新子任务达到 97.4%，时间推理子任务达到 89.5%。但策略检索延迟为 5-6 秒/查询，且仍处于研究阶段，生产就绪度较低。

## 核心能力

| 能力 | 说明 |
|------|------|
| 三重索引架构 | 主抽象（6-8 词短语）+ 记忆值（完整原始信息）+ 线索锚点（多路径访问标签） |
| 解耦存储与检索 | 记忆内容保持丰富，独立轻量级结构层负责索引，记忆值从不直接被检索 |
| 策略驱动检索器（MDP） | 将检索形式化为马尔可夫决策过程，支持 Refine/Expand/Stop 三种动作 |
| 多步迭代检索 | 重新表述查询、扩展搜索范围、主动停止，实现深度检索 |
| 强化学习蒸馏 | 检索策略可蒸馏到小型模型（Qwen-2.5-3B 验证，GRPO 训练后准确率 0.841） |
| 理论统一性 | RAG 和知识图谱检索都是 Memora 的特例，存在性证明其超越两者 |

## 技术架构

Memora 的三重索引架构是核心设计：第一层主抽象（Primary Abstraction）为 6-8 词短语，捕捉记忆核心本质，用于嵌入和相似性搜索，作为更新和聚合的单元——新信息合并到同一个主抽象下而非碎裂；第二层记忆值（Memory Value）保留完整原始信息（时间线、对话记录、决策过程），从不直接通过内容被检索，只能通过主抽象或线索锚点到达；第三层线索锚点（Cue Anchors）从记忆值中提取多个短标签，提供同一记忆的不同访问路径。

检索流程采用策略驱动的 MDP 形式化：查询先语义匹配主抽象/线索锚点生成候选记忆集，然后策略决定执行 Refine（重新表述查询）、Expand（扩展到关联记忆）或 Stop（信息足够停止），最终返回记忆集注入 LLM 上下文。消融实验表明，移除线索锚点后策略检索的优势消失——优势来自锚点提供的多路径导航而非策略本身的复杂性。

上下文粒度实验显示，原始片段情景+事实（LLM-Judge 0.863）优于摘要情景+事实（0.838）和仅事实（0.833），证实情景记忆与事实记忆存在互补效应。

## 适用场景

- 长期协作 Agent（需跨数周/数月保留上下文的 AI 助手）
- 多跳推理任务（需从多个角度检索信息的复杂查询）
- 半结构化信息处理（项目时间线、决策过程等非标准化信息）
- 知识更新频繁场景（需准确追踪信息变更）

## 局限性

- 策略检索延迟高：5-6 秒/查询，不适合实时交互
- 生产就绪度低：仍为研究代码，未集成到 Microsoft 365 Copilot
- 记忆构建成本：每个记忆需生成主抽象和多个线索锚点
- 缺少遗忘机制：何时应删除或降低旧记忆权重未解决
- 治理与合规缺失：缺乏审计日志、访问控制等企业级特性
- 缺乏实际部署案例：暂无企业级生产部署的公开案例
