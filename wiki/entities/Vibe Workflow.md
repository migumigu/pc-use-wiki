# Vibe Workflow

> 一种自然语言驱动的工作流编排范式，一个 Agent 节点可替代约 20 个传统工作流节点

## 核心概念

Vibe Workflow 是由 [[Refly.AI]] 提出的新一代工作流概念，核心思想是：

1. **自然语言描述**：用一句话描述需求，系统自动生成工作流
2. **Agent 节点**：用单个智能 Agent 节点替代多个传统 API 节点
3. **柔性执行**：允许一定的误差（70% 准确即可接受），通过多次交互优化
4. **动态调整**：工作流可根据执行结果自动调整

## 与传统工作流对比

| 维度 | Vibe Workflow | 传统工作流（n8n/Dify） |
|------|---------------|------------------------|
| 描述方式 | 自然语言 | 拖拽/编码 |
| 节点类型 | Agent 节点（智能） | API 节点（固定） |
| 容错能力 | 高（70% 即可） | 低（必须精确） |
| 灵活性 | 高（动态调整） | 低（固定流程） |
| 使用门槛 | 低（非技术人员） | 高（需要编码） |

## 适用场景

- 内容创作自动化
- AI 创意原型快速验证
- 非技术人员的工作流搭建
- 需要快速迭代的探索性任务
