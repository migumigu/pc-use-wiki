# Agent 长时记忆系统技术分析报告 v1.1

> Agent 长时记忆已进入"范式分化"阶段，Mem0、Memora、Eigent 三者分别占据事实提取、解耦索引和多 Agent 执行的生态层次，不存在银弹，只有场景匹配度。

## 关键信息

本报告聚焦三个代表性项目：Mem0（事实提取范式，41K-60K+ Stars，生产就绪度最高）、Memora（微软 ICML 2026，解耦存储与检索的范式创新者，理论突破最深）、Eigent（多智能体桌面工作流平台，5K-10K Stars，记忆系统的消费端载体）。三者分别占据 Agent 记忆生态的不同层次——Mem0 是当前的"事实提取王者"，Memora 是理论前沿的"导航式检索"突破者，Eigent 则展示了记忆系统在桌面级多 Agent 协作中的实际消费场景。

Agent 长时记忆系统的核心架构围绕三个基本问题展开：存什么（记忆粒度）、怎么找（检索策略）、何时忘（遗忘机制）。技术栈分为六层：记忆提取层（LLM 单次/多次调用）、索引层（向量/三重/图索引）、检索层（语义/BM25/MDP策略）、存储层（Qdrant/Neo4j/PostgreSQL）、集成层（MCP/SDK/REST API）、消费层（Desktop/Web/编码助手 Agent）。核心发现是三个项目均未实现自动遗忘机制，记忆只增不减。

能力对比显示：Mem0 在生产就绪度和检索延迟（<1s）上优势明显，但碎片化问题（651 条/对话）和叙事连贯性不足是根本局限；Memora 在上下文压缩效率（2.9K tokens）和检索深度（多步策略）上最优，但策略检索延迟高（5-6s）且仍是研究代码；Eigent 不涉及记忆功能，但在多 Agent 协作和桌面端执行上独树一帜。组合方案（Eigent 执行层 + Mem0 记忆层 + 本地模型）最适合企业级内部部署。

待验证问题包括：Mem0 新算法 vs Memora 的公平对比（不同模型栈和评估设置）、Memora GitHub 仓库地址、Eigent Stars 实时数据、Mem0 + Eigent 集成可行性、Memora RL 蒸馏实际收益（仅提升 0.5%）、Mem0g 图增强版本对比数据缺失，以及所有方案均缺少 6 个月以上长期部署的公开案例。
