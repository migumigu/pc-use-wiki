# 文件系统控制

> AI Agent 利用文件系统作为上下文、长期记忆、文件同步、锁机制和操作的技术体系

## 定义

<!-- confidence: INFERRED -->
文件系统控制是 AI Agent 通过文件系统实现以下能力的综合技术体系：
1. **上下文管理**：文件系统作为长期记忆和状态保存载体
2. **文件同步**：多设备间实时或双向同步
3. **文件锁机制**：进程间同步和并发控制
4. **文件操作**：复制、移动、删除、权限管理

## 核心观点

### 上下文管理视角

#### 1. 文件系统是 Agent 长期记忆的最佳载体

- **对话流只适合短暂的指令交互**：线性、嘈杂、包含过时纠错信息
- **文件系统才是智能体长期记忆和复杂推理的最佳载体**：单一真理来源、可恢复性、原子性、结构化

#### 2. Unix 理念的复兴

- **一切皆文件**：设备、管道、套接字都有文件描述符
- **统一接口**：open/read/write/close 简化编程模型
- **AI Agent 时代的应用**：文件系统被重新审视作为 Agent 基础设施

### 文件同步视角

#### 3. P2P vs 双向同步分化 <!-- confidence: EXTRACTED -->

- **Syncthing**（73K+ Stars）：P2P 无服务器架构，持续实时同步，强调安全和自动化
- **Unison**（25+ 年历史）：双向同步，冲突检测，精确规范，慢速链接优化

### 文件锁视角

#### 4. 跨平台锁机制成熟 <!-- confidence: EXTRACTED -->

- **filelock**（723K+ 使用）：提供 5 种锁类型（FileLock/SoftFileLock/ReadWriteLock/SoftReadWriteLock/AsyncFileLock）
- **平台支持**：Windows (msvcrt)、Unix (fcntl)、网络文件系统（SoftFileLock）

### 文件操作视角

#### 5. Python stdlib 局限性 <!-- confidence: EXTRACTED -->

- **shutil 无法复制完整元数据**：POSIX/Mac/Windows 都会丢失 owner/group/ACLs
- **Agent 集成风险**：使用 shutil 复制文件时需注意权限丢失，需补充 os.chown、ACLs API

## 核心原则

### 1. 文件即单一真理来源

对话历史是线性的、嘈杂的，包含大量过时的纠错信息。而文件是经过整理的、当前的「最新状态」。

AI 每次行动前，应该主要读取文件，而不是回溯几千行的聊天记录。

### 2. 状态显式化

通过外部化记忆（ToDo 文件）记录进度，即使重启电脑也能恢复。

### 3. 上下文窗口极简主义

只喂给 AI 当前步骤必要的信息。通过将信息拆分到 notes.md、plan.md 和 output.md，AI 在执行某一步时只需要读取相关片段。

### 4. 思考与行动分离

不让 AI 在一次回复中同时进行思考、规划、写代码：
1. 先在 Notes 上写下调研结果、架构思路
2. 确认无误后，再去修改正式的代码文件

### 5. 围绕 KV-Cache 进行设计

所有 Prompt 设计都要为缓存命中率让路：
- 保持前缀稳定
- 只追加不修改
- 确定性序列化

### 6. 掩码而非移除工具

保留所有工具定义，在解码阶段通过修改 Logits 屏蔽当前不合法的工具。

## 核心文件结构

### task_plan.md
记录当前任务的目标、拆解的步骤、已完成的进度、下一步要做什么。

### notes.md
存放调研资料、中间代码片段、临时的想法或长文本。

### [deliverable].md
最终生成的代码、文章或报告。纯净的输出结果，不包含思考过程。

## 与向量数据库的关系

不是替代关系，而是分工：
- **文件系统**：结构化、长期记忆、状态管理
- **向量数据库**：非结构化内容检索、RAG

## 应用场景

### 长时间任务管理
<!-- confidence: INFERRED -->
- 复杂项目的分步骤执行
- 多会话之间的状态延续

### 外部化记忆
<!-- confidence: EXTRACTED -->
即使关闭终端、重启电脑，AI 再次启动时读取文件，立刻知道进度条在哪里。

### 上下文污染防止
<!-- confidence: INFERRED -->
通过思考与行动分离，避免写到一半发现思路错了导致代码库被污染。

## 技术优势

1. **单一真理来源**：文件是经过整理的、当前最新状态
2. **可恢复性**：重启后读取文件即可恢复状态
3. **原子性**：操作具有原子性，易于版本控制
4. **结构化**：目录结构提供天然的组织方式
