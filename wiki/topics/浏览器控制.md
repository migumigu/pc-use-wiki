# 浏览器控制

> AI Agent 控制浏览器的完整技术栈

## 核心观点

浏览器控制是 AI Agent PC 控制的重要组成部分，涵盖：
- **传统方法**：通过 Playwright/Puppeteer 等框架编写代码控制浏览器
- **AI 增强**：通过 browser-use 等工具实现自然语言驱动的浏览器自动化
- **协议标准化**：MCP 协议实现跨平台的浏览器控制能力共享

## 关键概念

- [[browser-use]] — 自然语言驱动的浏览器自动化框架
- [[page-agent]] — 页面内 GUI Agent（阿里巴巴）
- [[agent-browser]] — AI 原生 Rust CLI 浏览器自动化（Vercel Labs）
- [[Playwright]] — 底层浏览器自动化引擎
- [[浏览器自动化]] — 通过程序控制浏览器的技术
- [[MCP]] — 浏览器控制的协议标准化

## 技术栈分层

### 系统基础层

- Chromium/Firefox/WebKit 浏览器引擎
- CDP（Chrome DevTools Protocol）协议
- 浏览器安全模型和沙箱机制

### 协议/接口层

- [[Playwright]] API
- CDP（Chrome DevTools Protocol）
- WebDriver 协议

### 工具实现层

- [[browser-use]] — 自然语言驱动（页面外）
- [[page-agent]] — 页面内 GUI Agent（阿里巴巴）
- [[agent-browser]] — AI 原生 Rust CLI（Vercel Labs）
- Playwright MCP — IDE 集成
- Puppeteer — Node.js 自动化

### Agent 集成层

- [[MCP]] — 工具调用标准化
- [[LLM]] — 任务理解和规划
- [[Agent]] — 自主执行和评估

## 未解决的问题

1. **反爬防护**：CAPTCHA 等防护措施难以绕过
2. **性能优化**：LLM 调用开销导致的性能问题
3. **稳定性**：页面变化导致的任务失败处理
4. **上下文限制**：长任务的 LLM 上下文限制
