# A2A Protocol 技术分析报告 v1.0

## 1. 执行摘要

A2A（Agent-to-Agent）Protocol 是 Google 于 2025 年 4 月发布的开放协议，现已捐赠给 Linux Foundation，旨在实现不同 AI Agent 之间的通信和互操作性。与 MCP（Model Context Protocol）形成互补：A2A 解决 Agent 之间的协作问题，MCP 解决 Agent 与工具/数据源的连接问题。

**核心价值**：
- 建立"智能体互联网"（Internet of Agents）
- 支持跨框架（LangGraph、CrewAI、Semantic Kernel 等）的 Agent 互操作
- 提供有状态的任务协作机制

### 2.3 核心组件

| 组件 | 作用 |
|------|------|
| **Agent Card** | JSON 元数据文档，用于 Agent 发现和能力描述 |
| **Task** | 有状态工作单元，跟踪长时间运行的操作 |
| **Message** | 单轮通信，包含角色和内容 |
| **Part** | 基本内容容器，支持 text/raw/url/data |
| **Artifact** | tangible 输出（文档、图像等） |

### 4.2 Agent 技术栈全景

| 层级 | 组件 | 协议/技术 |
|------|------|----------|
| **框架层** | LangGraph, CrewAI, ADK | Agent 开发框架 |
| **协议层** | MCP, A2A | 标准化通信协议 |
| **工具层** | Filesystem MCP, Browser Tools | 具体工具实现 |
| **模型层** | Claude, GPT, Gemini | LLM 智能核心 |

### 4.3 适用场景

- **多 Agent 协作系统**：需要多个专业 Agent 协同工作
- **跨平台 Agent 集成**：不同框架构建的 Agent 需要互操作
- **分布式 AI 系统**：Agent 运行在不同服务器上
- **企业级 Agent 应用**：需要安全性、认证、可观测性

### 4.4 不适用场景

- **单一 Agent 简单任务**：不需要多 Agent 协作
- **工具调用场景**：使用 MCP 更合适
- **实时性要求极高**：协议开销可能影响延迟
