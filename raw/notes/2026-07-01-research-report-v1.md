# 2026 年 AI Agent 新趋势研究报告 — Refly.AI + MCP-Link

> 生成日期：2026-07-01
> 来源：2 个（Tier1: 0, Tier2: 2, Tier3: 0）
> 报告版本：v1.0

## 1. 执行摘要

2026 年 AI Agent 生态呈现两大重要趋势：**工作流平台的 AI 原生化**和**MCP 生态的标准化扩展**。本次研究发现了两个代表性项目：

1. **Refly.AI**：全球首个 Vibe Workflow 开源平台，通过自然语言驱动的工作流编排，大幅降低 AI 工作流搭建门槛。一个 Agent 节点可替代约 20 个传统工作流节点，核心创新是"一句话生成工作流"。

2. **MCP-Link**：OpenAPI 规范到 MCP Server 的自动化转换工具，解决了存量 API 与 AI Agent 集成的核心痛点。支持 5 分钟内完成 API 到 MCP 的转换，预设了 10+ 主流服务的集成配置。

这两个项目共同反映了 2026 年 AI Agent 工具链的演进方向：**降低使用门槛**和**标准化生态扩展**。

#### MCP-Link 架构

### 2.2 技术栈分层

| 层级 | Refly.AI | MCP-Link |
|------|----------|----------|
| **系统基础层** | NestJS 后端、React 前端、多线程架构 | Go 语言、JSON-RPC over SSE |
| **协议/接口层** | MCP 协议、RESTful API | OpenAPI V3、MCP 协议 |
| **工具实现层** | 自由画布、多模型调度、RAG 检索 | OpenAPI Parser、API Adapter |
| **Agent 集成层** | Agent 节点编排、Vibe Workflow | MCP Server 自动生成 |

### 2.3 关键组件

#### Refly.AI 关键组件

1. **自由画布设计器**：基于 ReactFlow 的可视化工作流构建界面
2. **多线程会话系统**：每条对话保持独立上下文，支持并行任务处理
3. **多模型调度引擎**：支持 DeepSeek R1、Claude 3.5、Gemini 2.0、OpenAI O3-mini 等 13+ 模型
4. **RAG 检索模块**：支持从 GitHub、Medium、Wikipedia、Arxiv 等平台一键导入内容
5. **MCP 协议集成**：支持与外部工具和服务无缝集成

#### MCP-Link 关键组件

1. **MCP Router**：智能请求路由中心，自动根据 URL 参数和 Schema 类型路由请求
2. **OpenAPI Parser**：规范解析引擎，基于 OpenAPI V3 Schema 自动生成 MCP 接口
3. **API Adapter**：协议转换桥梁，支持 JSON-RPC over SSE 和 HTTP 协议
4. **预设配置库**：内置 Brave Search、DuckDuckGo、Notion、Slack、GitHub 等 10+ 服务的配置

### 3.2 局限性

#### Refly.AI 局限性

- **稳定性取舍**：为降低门槛，主动放弃部分准确性（70% 准确即可接受）
- **成本控制**：通过模型拆分控制成本，复杂任务可能需要多轮调用
- **内容生成聚焦**：主要面向内容生成场景，不适合需要 100% 准确的企业自动化

#### MCP-Link 局限性

- **OAuth 支持待完善**：目前不支持 OAuth 认证流程
- **资源型 API 处理**：对资源型 API 交互支持有限
- **MIME 类型支持**：对复杂 MIME 类型处理能力有待增强

### 3.3 已知问题

- Refly.AI：Agent 节点成本较高，但可通过工作流复用降低后续调用成本
- MCP-Link：需要确保 OpenAPI 规范质量，规范不完整会影响转换效果

### 4.2 适用场景

#### Refly.AI 适用场景

- 内容创作自动化（小红书文案、播客、报告、PPT）
- 智能数据分析与报告生成
- 企业知识库问答系统
- AI 创意原型快速验证
- 非技术人员的工作流搭建

#### MCP-Link 适用场景

- 自动化测试环境中的 API 集成
- AI Agent 与第三方服务的快速集成
- 现有 RESTful API 的 MCP 化改造
- 多服务复合应用的快速构建

### 4.3 不适用场景

#### Refly.AI 不适用场景

- 需要 100% 准确性的企业级自动化
- 复杂的业务流程编排（需要精确控制）
- 大规模数据处理任务

#### MCP-Link 不适用场景

- 无 OpenAPI 规范的 API（需要先编写规范）
- 需要复杂认证流程的 API（如 OAuth）
- 非 RESTful 的 API（如 SOAP）
