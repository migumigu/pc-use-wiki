## 1. 执行摘要

VulnClaw 是一个**AI 驱动的渗透测试 CLI 工具**，基于 LLM Agent + MCP 工具链 + 渗透 Skill 编排，实现自然语言驱动的全流程渗透测试。核心创新是"目标驱动求解引擎"——抛弃固定轮数工作流，以目标达成/探索前沿耗尽/安全预算为终止条件，自动收敛。支持 13 种 LLM Provider（OpenAI/DeepSeek/MiniMax 等），21 个渗透 Skill，4 个 MCP 服务，自动生成报告和 PoC。

**核心价值**：
- 说人话就能做渗透测试（学习成本低）
- 目标驱动求解，拒绝原地打转
- 证据级反幻觉闸门，杜绝凭空编造 flag
- 自动生成 Markdown 报告 + Python PoC

### 2.2 技术栈分层

| 层级 | 内容 | 关键技术 |
|------|------|----------|
| **系统基础层** | MCP 协议 | fetch/memory/chrome-devtools/burp |
| **协议接口层** | OpenAI 兼容协议 | Tool Calling、13 种 Provider |
| **工具实现层** | 21 Skill + 29 编解码 | pentest-flow、recon、vuln-discovery |
| **Agent 集成层** | 目标驱动求解引擎 | 黑板图、OODA 循环、反幻觉闸门 |

### 2.3 关键组件

1. **目标驱动求解引擎 (solve)**：黑板图状态空间搜索（Fact + Intent），拒绝原地打转
2. **证据级反幻觉闸门**：声称的 flag 必须在真实工具输出里逐字符出现
3. **OODA 循环**：Reason（读全图）→ Explore（领 Intent 执行）→ 写回 Fact
4. **自适应反思引擎**：失败自动归类，L0-L4 渐进升级 payload
5. **4 MCP 服务**：fetch（HTTP）、memory（状态）、chrome-devtools（浏览器）、burp（抓包）

## 3. 能力分析

### 3.1 支持的能力

| 阶段 | Skill | 能力 |

### 3.2 技术原理

**目标驱动求解引擎**：
- 黑板图：Fact（已确认事实）+ Intent（探索方向）
- OODA 循环：Reason → Explore → 写回 Fact
- 终止条件：目标达成 / 探索前沿耗尽 / 安全预算
- 结构上杜绝"原地打转"：Intent 领取一次、结论一次即标记 concluded/abandoned

**证据级反幻觉闸门**：
- 录制所有真实工具输出（HTTP 响应、python_execute 输出）
- 声称的 flag 未在真实输出里逐字符出现 → 判定幻觉、丢弃、标记 [未验证]
- Reason 宣布"目标达成"但无证据 → 拒绝完成、继续探索

**自适应反思**：
- 失败归类：环境限制 / 路径错误 / 参数错误 / 信息不足
- L0-L4 渐进升级：原始 → URL 编码 → 双写注释 → Unicode/hex → 多层混淆

## 4. 生态位

### 4.1 工具对比

| 维度 | VulnClaw | 传统工具（Nmap+Burp+SQLMap） |
|------|----------|----------------------------|
| **学习成本** | 低，说人话就行 | 高，需要记忆大量命令 |
| **自动化程度** | 高，全流程 AI 驱动 | 低，需要手动切换工具 |
| **工具数量** | 统一入口，11 MCP 服务 | Nmap+Burp+SQLMap+... |
| **报告生成** | 自动，Markdown + PoC | 手动，耗时耗力 |
| **上下文保持** | 跨周期状态记忆 | 每次任务重头来 |
| **模型支持** | 13 种，灵活切换 | 固定工具集 |
| **扩展性** | Skill + MCP，插件化 | 依赖工具更新 |

### 4.2 适用场景

**最适合**：
- CTF 竞赛（ctf-web、ctf-crypto、ctf-misc Skill）
- 已授权的渗透测试
- 红队演练
- 安全教学

**不适合**：
- 未经授权的渗透测试（违法行为）
- 生产环境直接测试（需隔离实验室）
- 需要精确控制的渗透（AI 可能产生误判）

### 4.3 发展趋势

**技术演进**：
- 从固定轮数工作流 → 目标驱动求解引擎（核心重构）
